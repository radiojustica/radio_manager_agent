# Implement Worker Status and Control API Endpoints Implementation Plan

## Overview
Expose the modular worker system's telemetry and control mechanisms via new FastAPI endpoints. This allows the frontend to monitor worker health and manually trigger execution cycles.

## Scope Definition (CRITICAL)
### In Scope
- Create `routers/workers.py` with endpoints for status and manual run.
- Register the new router in `main.py`.
- Integrate with `worker_manager_instance` for data and control.
### Out of Scope (DO NOT TOUCH)
- Modifying existing worker logic in `workers/*.py`.
- Changing the `WorkerManager` or `WorkerBase` core classes.
- Frontend implementation (UI).

## Current State Analysis
- `worker_manager_instance` is global and accessible.
- `WorkerBase.health()` provides the required telemetry.
- `WorkerManager.run_cycle(name)` handles execution and returns standardized results.
- Routers are located in `routers/` and registered in `main.py:153+`.

## Implementation Phases

### Phase 1: Backend Router
- **Goal**: Create the `routers/workers.py` file with the requested endpoints.
- **Steps**:
  1. [x] Create `routers/workers.py`.
  2. [x] Import `worker_manager_instance` from `worker_manager`.
  3. [x] Define `GET /api/workers/status`:
     - Iterate through `worker_manager_instance.workers.values()`.
     - Collect and return a list of `worker.health()` results.
  4. [x] Define `POST /api/workers/{name}/run`:
     - Validate if the worker exists.
     - Call `worker_manager_instance.run_cycle(name)`.
     - Return the execution result.
- **Verification**: Unit test the router logic (mocking the manager if necessary) or manual verification after Phase 2.

### Phase 2: Router Registration
- **Goal**: Make the new endpoints accessible via the main FastAPI app.
- **Steps**:
  1. [x] Import `workers` router in `main.py`.
  2. [x] Add `app.include_router(workers.router)` in the router registration section.
- **Verification**: Run the application and check `/docs` (Swagger UI) to confirm endpoints are visible.

### Phase 3: Verification & Validation
- **Goal**: Ensure the endpoints work as expected with live workers.
- **Steps**:
  1. [x] Start the application (`python main.py`).
  2. [x] Perform a `GET` request to `http://localhost:8000/api/workers/status`.
  3. [x] Perform a `POST` request to `http://localhost:8000/api/workers/GuardianWorker/run`.
  4. [x] Verify logs for worker execution.
- **Verification**: HTTP status codes 200 OK and expected JSON payloads.
