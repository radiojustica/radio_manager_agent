## Findings
1. **Existing Quarantine Logic**: 
    - Found in `services/curadoria_worker.py`.
    - Handles "Corrompido" (Mutagen failure) and "Inadequação" (Badwords).
    - Moves files to `D:\RADIO\QUARENTENA_TJ`.
    - Logs to `audit_quarentena.log`.

2. **Metrics Available**:
    - `librosa.load` is already used.
    - `rms` (Energy) and `spectral_centroid` (Brightness) are calculated.
    - `bpm`, `valence`, and `danceability` were recently added.

3. **Potential New Metrics**:
    - **Spectral Flatness**: `librosa.feature.spectral_flatness`. High flatness means the sound is noise-like (white noise/hiss). Low flatness means it's more tonal (music).
    - **Silence Detection**: If `rms` is consistently near zero, the file might be "dead air".
    - **Centroid Variance**: High variance in centroid might indicate unstable/distorted audio.

4. **Database State**:
    - `Musica` model lacks a `quarantine_reason` or `quarantine_date`.
    - `redflag` is a simple boolean.

5. **Integration Points**:
    - `services/curadoria_worker.py`: `processar_arquivo` needs to be updated with the new checks.
    - `workers/curadoria_worker.py`: Needs to capture and store the reason.
    - `api/manager.py` / `routers/acervo.py`: Needs endpoint to list quarantined files.

- **Energy < 2**: (As per PRD) Automatic quarantine.
- **Spectral Flatness > 0.5**: (Hiss/Noise) Flag for review.
- **Duration < 30s**: (Unless Jingle/Spot) Possible incomplete download.
