# Advanced Quarantine Implementation Plan

## Overview
Implement advanced audio quality checks and automated quarantine logic to ensure the station's acervo contains only high-quality, relevant tracks.

## Scope Definition
### In Scope
- Add `quarantine_reason` column to `Musica` model.
- Implement `Spectral Flatness` and `Silence` detection in `CuradoriaWorker`.
- Implement automatic quarantine for `Energy < 2`.
- Create API endpoint to list quarantined files and their reasons.
### Out of Scope
- Automatic deletion of quarantined files (safety first, even for a genius).
- Frontend UI for quarantine management (API only for now).

## Implementation Phases

### Phase 1: Database Schema Update
- **Goal**: Persist the reason why a track was flagged.
- **Steps**:
  1. [ ] Update `core/models.py` to add `quarantine_reason = Column(String, nullable=True)`.
  2. [ ] Update `core/database.py` to handle the migration in `init_db`.
- **Verification**: Run `python -c "from core.database import init_db; init_db()"` and check `radio_omni.db`.

### Phase 2: Advanced Audio Logic
- **Goal**: Calculate quality metrics.
- **Steps**:
  1. [ ] Update `services/curadoria_worker.py`: `analisar_acustica_completa` to return `spectral_flatness`.
  2. [ ] Update `services/curadoria_worker.py`: `processar_arquivo` to check for `Energy < 2`, `Flatness > 0.5`, or `Duration < 30s`.
  3. [ ] Set `status: "QUARANTINED"` and appropriate `motivo` if checks fail.
- **Verification**: Run `tests/test_audio_analysis.py` (need to create/update it).

### Phase 3: Worker & API Integration
- **Goal**: Store results and expose via API.
- **Steps**:
  1. [ ] Update `workers/curadoria_worker.py` to save `musica.quarantine_reason = resultado.get("motivo")`.
  2. [ ] Update `routers/acervo.py` to add `GET /api/acervo/quarantine` returning all `redflag=True` records.
- **Verification**: Call API via `curl` and verify JSON output.

## Verification Plan
- **Automated**: `pytest tests/test_audio_analysis.py`.
- **Manual**: Add a low-energy/noisy file to the acervo and run `CuradoriaWorker` manually.
