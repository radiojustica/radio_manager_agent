# Implementation Plan: Intelligent Playlisting with Weather-Mood Mapping

## Overview
Transform the static playlist generation into a dynamic, environmental-aware system. We will replace the heuristic-based `WeatherService` with actual data and integrate the `GestorFila` rules for better flow and anti-repetition.

## Scope Definition (CRITICAL)
### In Scope
- Upgrade `services/weather_service.py` to fetch real weather data for Natal/RN.
- Update `director/grade_rules.py` to enhance mood mapping and `GestorFila` logic.
- Integrate mood-based selection in `director/playlist_engine.py`.
- Implement reward logic for `PlaylistWorker`.
### Out of Scope (DO NOT TOUCH)
- Modifying the React frontend.
- Changing the database schema.
- Updating other workers unrelated to playlisting.

## Current State Analysis
- **WeatherService** (`services/weather_service.py:6`): Hardcoded time-based heuristics.
- **GradeRules** (`director/grade_rules.py:73`): Static `MOODS` dictionary.
- **GestorFila** (`director/grade_rules.py:118`): Already handles basic anti-repetition but lacks mood-integration depth.
- **PlaylistEngine** (`director/playlist_engine.py:101`): Already attempts to call `weather_service`.
