# Implement Worker Execution History in RewardStore Implementation Plan

## Overview
Enhance the `RewardStore` to maintain a persistent, rolling history of execution results for each individual worker. This ensures that even high-frequency workers don't drown out the history of others and provides better debugging visibility.

## Scope Definition (CRITICAL)
### In Scope
- Modify `RewardStore.record` to store a rolling list of the last 20 records per worker.
- Update `RewardStore.history` to utilize the per-worker history for faster and more reliable retrieval.
### Out of Scope (DO NOT TOUCH)
- Modifying the global history limit (stays at 1000).
- Changing `WorkerBase` or worker logic.
- Adding a UI for history visualization (API only).

## Current State Analysis
- Global history is capped at 1000 entries [L86-87].
- `RewardStore.history()` scans the entire global list to filter by worker [L105-111].
- Worker summary data only stores the `last_result` [L70-75].

## Implementation Phases

### Phase 1: Persistent Per-Worker History
- **Goal**: Update `record()` to maintain a rolling history list within each worker's data entry.
- **Steps**:
  1. [x] In `core/reward.py:record()`, initialize `worker_data["history"]` if it doesn't exist.
  2. [x] Append the new `record` to `worker_data["history"]`.
  3. [x] Prune `worker_data["history"]` to keep only the last 20 entries.
- **Verification**: Run a worker cycle multiple times and inspect `worker_rewards.json` to see the new `history` field under each worker.

### Phase 2: Optimized Retrieval
- **Goal**: Update the `history()` method to use the per-worker list when a `worker_name` is provided.
- **Steps**:
  1. [x] Refactor `RewardStore.history()` in `core/reward.py`.
  2. [x] If `worker_name` is provided, return the last `limit` entries from `self.data["workers"][worker_name]["history"]`.
  3. [x] Keep global history scanning as a fallback for the combined view.
- **Verification**: Call `history(worker_name="GuardianWorker")` and verify it returns correct results even if thousands of global cycles have passed.
